# Content Strategy

Last updated: 2026-02-27

## Objective
- Grow high-intent organic traffic and repeat usage for `emoj.ie` by publishing content that improves emoji discovery quality and copy workflow confidence.

## Audience Segments
- Daily chat/social writers who need fast copy outcomes.
- Creators and marketers who need meaning/context before posting.
- SEO users searching for intent-led terms (`[emoji] meaning`, `[topic] emojis`, `[site] alternative`).

## Pillars

### 1) Search Intent Education (Searchable)
- Goal: rank for practical "how do I find the right emoji" questions.
- Formats:
  - `/search/*` intent hubs (existing)
  - supporting blog posts under `/blog/*`
- Topics:
  - slang-to-emoji mappings
  - typo-tolerant search walkthroughs
  - emoji codepoint search guides

### 2) Workflow Optimization (Searchable + Shareable)
- Goal: convert first-time users into repeat users.
- Formats:
  - tutorials, checklists, short demos
- Topics:
  - favorites + recents usage patterns
  - keyboard-first copy loops
  - copy format choice (emoji/unicode/html/shortcode)

### 3) Comparison And Decision Content (Searchable)
- Goal: capture competitor-intent traffic and help evaluators choose quickly.
- Formats:
  - `/alternatives/*` pages (existing + expansion)
  - planned `/vs/*` and plural alternatives pages
- Topics:
  - switch triggers by user type
  - migration path content
  - fit-based recommendations (honest tradeoffs)

### 4) Emoji Confidence Content (Searchable)
- Goal: increase detail-page utility and reduce mis-use anxiety.
- Formats:
  - emoji detail page enrichments
  - short explainers for high-traffic emoji groups
- Topics:
  - meaning + usage examples
  - related emoji decision trees
  - ambiguity and context rules

## 12-Week Publishing Plan

### Weeks 1-4: Foundation
1. Publish 4 workflow posts:
   - fast emoji search patterns
   - copy-format guide
   - favorites/recents setup
   - typo-proof searching
2. Expand alternatives IA brief (plural + vs templates).
3. Add internal links from workflow posts to `/search/*`, `/tag/*`, `/emoji/*`.

### Weeks 5-8: Expansion
1. Publish 4 intent-led category guides:
   - emojis for announcements
   - emojis for customer support
   - emojis for product launches
   - emojis for community replies
2. Launch first two new comparison templates (`/vs/*`).
3. Add FAQ sections to top comparison pages and validate schema.

### Weeks 9-12: Authority
1. Publish 4 data-backed recap posts:
   - most searched emoji intents
   - no-result query themes and fixes
   - copy-format adoption patterns
   - theme/device behavior insights
2. Refresh top 10 pages using analytics and on-page engagement.
3. Produce an internal "what changed" report for next cycle priorities.

## Editorial Guardrails
- Every page must have clear user utility beyond keyword targeting.
- No thin pages: each new page must include concrete guidance, examples, and next actions.
- Use consistent naming and route hygiene (canonical + breadcrumb + schema).
- Prefer specific claims and measured statements over vague adjectives.

## Distribution Linkage
- Every new article gets:
  - one X post
  - one LinkedIn post
  - one community repost (Reddit or niche forum where appropriate)
- Each distribution artifact should route readers to one owned destination.

## Measurement Plan
- Primary metrics:
  - organic entrances per route family (`/emoji`, `/search`, `/tag`, `/alternatives`)
  - copy events per session
  - repeat sessions (favorites/recents interactions)
- Event signals:
  - `search`, `search_no_results`
  - `copy`, `filter`
  - `favorite_add`, `favorite_remove`, `recent_reuse`
  - `share`
- Weekly review:
  - top no-result query shapes
  - pages with high entrances + low copy actions
  - pages with strong copy actions but low retention (candidate for favorites CTA tuning)
