```css
@media (prefers-reduced-motion: reduce) {

HOWEVER, keep the existing breakpoint overrides for category hub pages where `.page-home .panel-grid .panel-card` and `.panel-grid-balanced .panel-card` need specific sizing. The auto-fill approach may require testing. If it looks wrong, keep the explicit breakpoints but increase from 6 max to accommodate larger tiles. Actually -- the panel-grid is used for BOTH category card grids (home page) AND emoji tile grids (EmojiGrid.svelte). Category cards need different column behavior than emoji tiles. To be safe: leave `.panel-grid` breakpoints as-is (they work for category cards). The emoji tile grid in EmojiGrid.svelte will get its own grid class in Plan 02. Do NOT change the existing `.panel-grid` grid-template-columns. Instead, just ensure the new token values exist for Plan 02 to consume.
